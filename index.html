使用 hexo+github 搭建个人博客简记

By 赵成龙

Published 2018-04-11

总体顺序

  1. 安装 node.js。 
    网址如下[node.js](https://nodejs.org/en/)。静默安装即可，完成后可以在命令行中使用 node -v 检测，如果出现版本信息，表示安装成功。
  2. 安装Git。 
    网址如下[Git](https://git-scm.com/download/win)。静默安装即可，
完成后可以在命令行中使用 git --version 检测，如果出现版本信息，表示安装成功。
  3. 安装hexo。 
    桌面右键鼠标，点击Git Bash Here，输入以下npm命令即可安装。
npm install hexo-cli -g   （安装hexo）
npm install hexo-deployer-git --save   （安装hexo部署到git page的deployer插件）
  4. hexo初始化。 
    在盘中任意位置创建一个文件夹（该文件夹用于保存博客相关的信息），比如 d:\hexo\blog。
在该文件夹下打开命令行或者使用Git Bash Here 输入 hexo init命令。 
等待命令完成后 依次输入 hexo g (生成静态文件，会在当前目录下生成一个新的叫做public的文件夹)
hexo s(启动本地web服务，用于博客的预览). 
等待命令执行完成后 在浏览器中访问https://localhost:4000/ 此时可以看到一个博客的系统网页。
  5. 将博客部署到Githubpage上。
    (1)在github中创建一个新的仓库（仓库名称为:yourGithubName.github.io,比如你的github名是sky,那么仓库名称就是sky.github.io).并在下方的相关选项中选择public和Initialize this repository with a README。
(2)配置SSH密钥。（此处创建的密钥用于添加到github中，实现本地git和网络中自身github账户的通信验证。）
   1.首先检查是否存在SSH密钥(keys)，步骤如下，任意位置打开Git Bash Here,输入以下命令： cd ~/.ssh
   如果存在直接进入3。
  （这一步主要是为了提醒如果你之前设置过密钥，可以直接去使用而无需去操作步骤2，
   如果从来没有进行锅步骤2，本步骤可以省略，直接进入步骤2.）
   2.创建SSH密钥。 在Git Bash Here命令中输入以下命令 
      $ git config --global user.name "userName"//用户名
      $ git config --global user.email "userEmail"//填写自己的邮箱
      ssh-keygen -t rsa -C "userEmail"  输入命令后等待执行，按照提示按回车键即可，无需输入任何密码。
      之后在C:\Users\yourcomputername\.ssh可以发现一个id_rsa的pub格式文件，该文件内部保留的就是我们需要的SSH密钥。
   3.将2中创建（或者是原本存在的）的文件中的密钥添加到github账户中。 在github账户中点击图像（右上角的图像），选中Settings.
  （在一些老一点的教程中会让你使用自己当前创建的库中的Settings,自己尝试了一下并没有效果。）
   选中SSH and GPG keys,点击上方的new SSH key,在点击后的页面中 title任取，将密钥添加到key选项中。保存即可。
   4.配置测试。
   在任意位置桌面右键鼠标，点击Git Bash Here  输入以下命令 ssh -T git@github.com 
   (1)如果提示你输入yes/no? 输入yes即可。那么配置通过。
   (2)如果提示连接超时，那么问题在于.ssh文件夹中缺少配置文件。
   注意配置文件的保存格式不是txt,也不是xml。
   在用notepad++编辑时另存为: (文件名：config 保存类型：All types(*.*) ) 否则无效。
  6. 更换博客主题。
    (1) 首先选择你喜欢的博客主题。[主题网站](https://git-scm.com/download/win)。
(2) 将你喜欢的主题库clone到本地。 点击主题后进入不同的网站，按照网站下方的提示进行相关操作。
以[此处为例](https://github.com/AlynxZhou/hexo-theme-aria/blob/master/README.zh_CN.md)
网站提示操作如下  git clone https://github.com/AlynxZhou/hexo-theme-aria themes/aria。
我们需要将其修改为：git clone https://github.com/AlynxZhou/hexo-theme-aria 本地的主题库
git clone https://github.com/AlynxZhou/hexo-theme-aria d:\hexo\blog\themes/aria
(3) 修改配置文件。选中本地blog文件夹下的_config.yml文件。
将其中 theme: xxx 替换为(2)中的主题即可，比如此处是 aria。
  7. 修改部署
    配置文件中部署只是针对本地web,配置到githubpage中需要做如下修改。将_config.yml文件中的deploy替换如下
deploy:
type: git
repo: git@github.com:zcl2015/zcl2015.github.io.git
branch: master
其中上方的repo是yourGithubName.github.io仓库中Clone or download中的内容。替换即可。
  8. 运行测试。
    hexo d -g 运行后 访问yourGithubName.github.io应该可以访问博客页面了。

war包和web.xml的部分讲解

By 赵成龙

Published 2018-05-29

此处记录Servlet中的war包以及web.xml中的部分内容

关于WAR包

什么是WAR包

Web存档(war)文件包含Web应用程序的所有内容。它减少了传输文件所需要的时间。一个war(web archive)文件包含一个web项目的文件。它可能有servlet，xml，jsp，image，html，css，js等文件。

WAR包的优点

war文件将所有文件合并为一个单位。 所以在将文件从客户端传输到服务器时需要更少的时间。

WAR包的导出方法

此处使用Eclipse导出War文件。在项目名称上点击右键，选择：Export->War File，在弹出的对话框选选保存的位置。然后将该文件复制到tomcat下面的webapps目录下。在bin文件夹下启动项目，然后在浏览器中访问（访问地址与eclipse中的访问地址一致。）。

关于web.xml中的welcome-file-list

welcome-file-list简介

在web.xml文件中的web-app块的welcome-file-list子元素用于定义欢迎文件列表。它的子元素是welcome-file，用于定义欢迎文件(即默认打开的页面)。
如果不指定任何文件名，欢迎文件是服务器自动调用的文件。例如如果在浏览器中只是输入了项目名称，但是并没有指定相关的URL,那么此时就会自动跳转到欢迎文件。

服务器查找欢迎文件顺序如下（todo）

  1. web.xml文件中的welcome-file-list指定的文件
  2. index.html
  3. index.jsp

如果在web.xml中指定了welcome-file，并且所有文件index.html和index.jsp都存在，那么优先级将转到welcome-file。
如果web.xml文件中不存在welcome-file-list项，那么优先级到index.html文件，然后是index.jsp文件。

GenericServlet类简介

Servlet接口为所有servlet提供常见的行为。
GenericServlet类实现了Servlet，ServletConfig和Serializable接口。它提供除了service方法之外，还实现了这些接口的所有方法。
GenericServlet类可以处理任何类型的请求，因此它与协议无关。
可以通过继承GenericServlet类并提供服务方法的实现来创建通用的servlet。

Spring由类型(Type)自动装配

在Spring中，“类型自动装配”的意思是如果一个bean的数据类型与其它bean属性的数据类型相同，将自动兼容装配它。
在本例中，“persion” bean以“ability”类数据类型作为属性，Spring会找到与ability类相同的数据类型，并自动装配它的Bean。如果没有匹配找到，它什么也不做。

  • 启动自动装配模式，需要确保只有Bean 只有一个唯一的数据类型声明。
  • 如果没有任何匹配，类数据类型将会用默认值null。

关于上方两种配置方式的说明：
上方中的i行并没有设置id,这说明@Autowired自动装配规则是基于byType。在之前基于xml的例子中需要保证相关的bean的唯一性（bean不存在或者bean只有一个），而此处有不同。体现在以下两个方面:

  1. 相关的bean的个数一般不能为0，因为默认情况下，@Autowired将执行相关检查，以确保属性已经装配正常。当Spring无法找到匹配的Bean装配，它会抛出异常。要解决这个问题，可以通过 @Autowired 的“required” 属性 设 置为false来禁用此检查功能。
    基于此，可以默认使用 @Autowired(required = false)来避免检查。
  2. 相关bean的个数可以是多个（这也是这种配置方式由于xml的地方）。添加@Qualifier即可（往下看）。

补充说明

关于@Qualifier
@Qualifier注解用来控制bean应在字段上自动装配。该注解不支持构造方法（但是如果java类中存在构造方法时需要将@Autowired用在构造方法上，因此当使用@Qualifier注解时不要出现构造方法）
java代码修改如下：

package com.yiibai.host;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
public class Person
{
@Autowired
@Qualifier("hello")  // pos1
private Ability ability;

public Ability getAbility() {
    return ability;
}

@Autowired
@Qualifier("hello")  // pos2
public void setAbility(Ability ability) {
    System.out.println("autowiring by type");
    this.ability = ability;
}
// @Autowired
// @Qualifier("hello")
/*public Person(Ability ability) {
    System.out.println("autowiring by constructor");
    this.ability = ability;
}*/    // pos3

@Override
public String toString() {
    return "Person [ability=" + ability + "]";
  }
}

关于上方的说明：

  • 注解的位置可以是pos1或pos2,使用@Qualifier注解时 pos3的整段代码不要出现。

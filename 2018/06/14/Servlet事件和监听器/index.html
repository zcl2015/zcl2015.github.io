Servlet ServletContextEvent事件

什么是Servlet ServletContextEvent事件？

在之前学习ServletContext时我们知道ServletContext对象由Web容器在部署项目时创建。而当Web应用程序部署在服务器上时，会通知ServletContextEvent事件。
因此如果要在部署Web应用程序时执行某些操作，例如创建数据库连接，创建项目的所有表等，可以通过实现ServletContextListener接口的方式完成。

ServletContextEvent ServletContextListener 介绍

ServletContextEvent类中只定义了一个方法：

public ServletContext getServletContext() - 返回ServletContext的实例。

在ServletContextListener接口中声明了两种方法，需要自定义实现。

public void contextInitialized(ServletContextEvent e): 当应用程序部署在服务器上时被调用。

public void contextDestroyed(ServletContextEvent e): 当应用程序从服务器取消部署时被调用。

HttpSessionEvent

什么是HttpSessionEvent？

会话对象更改时通知HttpSessionEvent，此事件的相应侦听器接口是HttpSessionListener。
我们可以在这个事件上执行一些操作，例如：计数统计当前登录的用户，记录登录时间，注销时间等用户详细信息。

如果对此处内容有所遗忘。可以参考上一节的Servlet会话追踪

HttpSessionListener接口介绍

在HttpSessionListener接口中声明了两种方法，需要由servlet程序员来实现。

public void sessionCreated(HttpSessionEvent e)：在创建会话对象时被调用。
public void sessionDestroyed(ServletContextEvent e)：当会话无效时被调用。
